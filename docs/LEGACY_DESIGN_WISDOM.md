# 老游戏设计智慧结构

这份文档写的是“老游戏为什么会显得聪明”。

它不是剧情总结，也不是攻略摘要，而是把这类 90 年代日式策略游戏里最值得借鉴的设计结构抽出来。

## 1. 设计不是线，而是结构

这类游戏看起来像故事，其实底层是结构系统。

它们不是靠一条线把玩家拖到底，而是靠：

- 身份
- 地点
- 资源
- 关系
- 任务
- 事件
- 旗标

把世界拧成一个会自己变化的网络。

## 2. 最关键的设计原语

### 2.1 身份驱动

角色不是固定头像，而是状态体。

身份变化会改变：

- 可进入的地点
- 可接受的任务
- 可结识的人物
- 可触发的分支

### 2.2 地点驱动

地点不是背景，而是触发器。

一个港口、教会、酒馆、造船厂、宫廷，都会改变玩家可见的事件和关系。

### 2.3 资源驱动

钱、船、货、情报、器材、资助，这些不是库存装饰，而是分支条件。

资源决定：

- 能不能出发
- 能不能继续
- 能不能换线
- 能不能从被动变主动

### 2.4 关系驱动

人物不是孤立 NPC，而是关系网节点。

一个资助人、教官、对手、同行者、家族成员，都可能是跨线桥。

### 2.5 事件回流

老游戏常常让同一个目标在多个线里反复出现。

这样做的效果不是重复，而是让世界有“回声”。

### 2.6 跨线交叉

最聪明的地方不是单线本身，而是线与线之间的交叉。

例如：

- 一个资助人同时连接两条线
- 一个远方目标串起多个主人公
- 一个人物把海盗线和军方线接在一起

### 2.7 有限信息下的世界感

很多信息不是一次给完，而是通过港口传闻、任务提示、对话碎片慢慢露出来。

这种做法会让世界显得更大，也更像“真有一个外部世界在运转”。

## 3. 为什么这种结构今天还值得学

因为它们不是依赖大制作堆出来的复杂，而是依赖结构设计。

这对今天很重要：

- AI movie 需要分叉和状态。
- 交互叙事需要关系和回流。
- 策略游戏需要资源和地点。
- 产品设计也需要状态驱动，而不是死流程。

## 4. 老游戏设计智慧的核心结论

一句话说完就是：

**真正聪明的老游戏，不是写了很多剧情，而是把世界做成一个会因为身份、资源、关系和地点变化而自己分叉的系统。**
