# 逆向成果账本

这份文档把现有逆向成果分成两本账：

- **方法账**：我们是怎么把东西拆出来、索引出来、验证出来的。
- **设计账**：我们从老游戏里看到了什么设计结构、什么世界组织方式。

它的作用是把“做了很多东西”重新收束成两条清楚的线，方便后面继续做样板和总结。

## 1. 方法账

方法账关心的是流程和证据，不关心剧情本身好不好看。

### 1.1 文件盘点

先把原始文件分层：

- 压缩资源：`.lzw`
- 文本容器：`Message.dat`、`Snr*.mes`
- 数据表：`Windcur.dat`、`Za_dat.dat`、`Transit.dat`、`Monster.dat`
- 主程序：`Main.exe`
- 主数据：`KOUKAI2.DAT`

这一步的意义是先搞清楚“什么东西在哪一层”，不要一上来就解释剧情。

### 1.2 资源抽取

先把能看的东西抽出来：

- 头像、港口图、世界图、船图、角色图
- 文本原文
- 数据表 JSON

对应产物主要都在：

- `output/contact_*.png`
- `output/game_data/`
- `docs/解出的游戏文本.md`

### 1.3 结构索引

接着建索引，把原始材料整理成可以查的形式：

- 资源索引
- 文本索引
- 角色索引
- 地点索引
- 事件索引

这一步的目标不是下结论，而是让材料能被重新组合。

### 1.4 拓扑分析

再把结构压成图：

- `docs/游戏逻辑说明.md`
- `docs/six_storylines_overview.png`
- `docs/character_relations.png`
- `docs/location_network.png`
- `docs/event_state_machine.png`
- `docs/event_chain_graph.png`

这部分已经把六条主人公线、人物交叉、地点网络和状态流转挂起来了。

### 1.5 动态验证

最后再用调试和追踪确认猜测：

- `scripts/sndt_trace_runner.py`
- `output/sndt_trace/`
- `docs/archive/SNDT_*`

这一层的目的，是把“看起来像”变成“有证据支持”。

## 2. 设计账

设计账关心的是结构含义，不关心某条文本具体怎么说。

### 2.1 六条主人公线

现在已经能稳定读出六条线的功能分工：

- `Snr1`：贵族少年出海线
- `Snr2`：西班牙军官复仇线
- `Snr3`：国家战争 / 私掠线
- `Snr4`：地理学家制图线
- `Snr5`：资助 / 冒险 / 寻宝线
- `Snr6`：贸易 / 发家 / 寻妹线

### 2.2 核心交叉点

设计上最重要的不是单线，而是交叉：

- `公爵夫人`：把 `Snr1` 和 `Snr5` 连起来
- `约翰·法雷尔`：把 `Snr1`、`Snr4`、`Snr5`、`Snr6` 串成全局目标轴
- `卡特琳娜`：把 `Snr1` 和 `Snr2` 接起来
- `梅尔卡特`：把地理线和寻宝线接起来
- `夏洛克 / 皮耶德 / 出納拉迪楠`：把资金线和贸易线接起来

### 2.3 设计原语

从现有材料里已经可以抽出几类设计原语：

- 身份驱动
- 地点驱动
- 资金驱动
- 关系驱动
- 事件回流
- 跨线交叉
- 世界目标驱动

这些就是后面要写进新游戏和 AI movie 的结构库存。

## 3. 现在的结论

这份项目不是只在“破解内容”。

它同时在做两件事：

- 记录我们是怎么把旧游戏拆开的。
- 记录我们从旧游戏里学到了什么设计结构。

所以后面所有新文档，都应该尽量落回这两本账。
