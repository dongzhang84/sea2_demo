## 1. Phase 3 的目标

Phase 3 要做的不是继续挖原作，而是把已经恢复出来的拓扑，收束成一份可执行的设计稿。

这份设计稿要回答四个问题：

1. 这个游戏的核心循环是什么。
2. 它的世界状态怎么分叉。
3. 人物、地点、资源、任务如何彼此影响。
4. Phase 4 真正实现时，应该先做什么，不应该做什么。

换句话说，Phase 3 的产物是：

```text
逆向结果 -> 可实现规格
```

## 2. 从原作拓扑提炼出的设计结论

当前已经恢复的拓扑说明，这个游戏不是单线剧情，而是一个状态驱动的分叉世界。

已确认的结构信号：

- 六条主人公线并行存在。
- 线与线之间不是独立的，而是通过人物、地点、状态和共同文本反复交叉。
- 最强的桥节点不是地图上的一个点，而是跨线的人物与目标。
- 事件不是单句对话，而是“状态推进 + 文本输出 + 下一跳选择”的组合。

当前最有价值的拓扑结论是：

- `Snr1` 是贵族少年出海线，负责“被逐出家门 -> 筹资 -> 出航 -> 约翰/卡特琳娜”。
- `Snr2` 是军官复仇线，负责“失踪调查 -> 军方阻止 -> 私掠/复仇”。
- `Snr3` 是王命海军线，负责“国王召见 -> 组舰队 -> 战争扩散”。
- `Snr4` 是制图/知识线，负责“世界地图 -> 劳拉寻亲 -> 东亚/南美”。
- `Snr5` 是资助/寻宝线，负责“欠债 -> 公爵夫人资助 -> 黄金之国”。
- `Snr6` 是贸易/发家/寻妹线，负责“穷困 -> 借债/贸易 -> 寻莎莎/同盟港”。

这些线之间的桥节点已经被明确看出来：

- `公爵夫人` 连接 `Snr1` 和 `Snr5`
- `约翰` 连接 `Snr1`、`Snr4`、`Snr5`
- `卡特琳娜` 连接 `Snr1` 和 `Snr2`
- `梅尔卡特` 连接 `Snr4` 和 `Snr5`
- `法布利斯` 连接 `Snr4` 和 `Snr5`
- `夏洛克 / 皮耶德 / 出納拉迪楠` 连接 `Snr5` 和 `Snr6`
- `劳拉 / 薩莎` 把 `Snr4` 和 `Snr6` 重新拉回到家庭/寻亲线

这说明新游戏的关键不是“塞很多任务”，而是做一个会自己分叉的世界。

## 3. 新游戏的设计原则

### 3.1 世界应当是状态驱动的

核心规则不是“走到哪触发哪”，而是：

```text
人物状态 + 地点状态 + 资源状态 + 关系状态 + 任务状态
-> 可触发事件
```

### 3.2 分支必须可回流

好的分支不是一次性分叉，而是：

- 先分出去
- 中间因资源、人物或地理条件重新相交
- 再分出去

这比纯树状剧情更接近当前拓扑里看到的结构。

### 3.3 桥节点优先

Phase 4 不能先做大世界铺量，而应该优先做桥节点。

原因很简单：

- 桥节点最能体现系统性的交叉。
- 桥节点最能证明这个游戏不是线性任务串。
- 桥节点最能复用原作拓扑里已经恢复出来的结构。

## 4. 核心玩法循环

新游戏的主循环建议如下：

```text
接任务 / 接触人物
-> 在港口整理资源
-> 选择船只与补给
-> 航行到下一个地点
-> 遭遇事件 / 贸易 / 战斗 / 发现
-> 更新人物状态、地点状态、资源状态
-> 触发新的桥节点
```

玩家每一轮做的不是单纯买卖，而是在推进一个世界状态机。

### 4.1 玩家每轮关心的资源

- 资金
- 船只与载货量
- 船员与可靠伙伴
- 声望 / 王命 / 任务旗标
- 地图知识
- 当前港口关系

### 4.2 玩家每轮做的决定

- 去哪一个港口
- 接不接某个人的请求
- 把资源投到贸易、修船还是剧情
- 是否接受国家任务或私人委托
- 是否进入高风险航路

## 5. 六线拓扑如何变成设计内容

### 5.1 Snr1：贵族少年线

设计作用：

- 教玩家“被迫离开起点后，如何重新建立出海能力”。
- 把 `公爵夫人`、`洛克`、`约翰`、`卡特琳娜` 这些桥人物介绍出来。

设计重点：

- 家门外身份
- 筹资
- 造船
- 初次出海
- 约翰线索
- 海上政治卷入

### 5.2 Snr2：军官复仇线

设计作用：

- 提供“调查 -> 阻力 -> 体制外行动”的叙事模板。

设计重点：

- 失踪事件
- 司令部阻断
- 港口情报循环
- 卡特琳娜插入
- 私掠式解决方式

### 5.3 Snr3：王命海军线

设计作用：

- 作为“国家级任务线”的模板。
- 给系统一个更高层级的权力结构。

设计重点：

- 国王召见
- 许可
- 组舰队
- 对抗西班牙

### 5.4 Snr4：制图 / 知识线

设计作用：

- 把航海从“打仗和赚钱”扩展成“知识和发现”。

设计重点：

- 梅尔卡特
- 世界地图
- 劳拉寻亲
- 黄海 / 日本
- 南美 / 黄金之国

### 5.5 Snr5：资助 / 寻宝线

设计作用：

- 把“社交关系 -> 资源 -> 远行 -> 寻宝”串起来。

设计重点：

- 欠债
- 公爵夫人资助
- 冒险报告
- 金质奖章
- 黄金之国 / 圣人手杖
- 法布利斯家族线

### 5.6 Snr6：贸易 / 发家 / 寻妹线

设计作用：

- 这是最适合做现代化系统的线，因为它天然和经济、关系、目标回流绑在一起。

设计重点：

- 穷困起步
- 修船
- 借债
- 贸易
- 投资
- 寻找薩莎
- 奥斯曼任务 / 同盟港

## 6. 适合做成 MVP 的范围

Phase 3 不应该把整套世界一次性写满，而应该定义一个能在 Phase 4 落地的 MVP。

### MVP 推荐范围

建议先做一个“小世界”，但必须保留拓扑本质：

- 3 个港口
- 1 条主线
- 2 个桥节点
- 1 个贸易循环
- 1 个航海循环
- 1 个状态回流

### 推荐的垂直切片

最适合做成第一版垂直切片的是：

- `Snr1` 的起点
- `Snr5` 的资助/寻宝桥
- `Snr6` 的贸易/借债桥

原因：

- 它们最能体现“身份 -> 资金 -> 船 -> 外出 -> 回流”。
- 它们最接近玩家第一小时需要理解的系统。
- 它们最容易形成闭环，而不是只做一个看不完的剧情开头。

### MVP 不要先做的东西

- 不要先做 6 线全量剧情。
- 不要先做完整世界地图。
- 不要先做全部海战细节。
- 不要先做所有 NPC。

先做的是：

```text
小世界 + 状态驱动 + 事件桥 + 贸易闭环
```

## 8. 数据模型建议

Phase 4 的实现数据，不要直接沿用原作文件名，而要换成新游戏内部结构。

### 推荐 schema

```yaml
world:
  ports: []
  routes: []
  regions: []

characters:
  - id: duchess
    role: bridge
    lines: [snr1, snr5]
    states: []

events:
  - id: event_snr1_01
    line: snr1
    triggers: []
    effects: []

state:
  money: 0
  ship: {}
  fame: 0
  flags: []
```

### 关键原则

- 人物、地点、事件、状态必须拆开。
- 事件必须能挂回角色和地点。
- 状态必须能被 UI 读出来。
- 数据必须可编辑，不要写死在代码里。

## 9. 设计风格建议

这个项目最适合的气质不是“复制旧游戏”，而是：

- 海洋冒险
- 轻策略
- 关系驱动
- 事件分叉
- 世界知识感

视觉上可以保留航海时代的庄重感，但系统上要更现代：

- 可检索
- 可回流
- 可重玩
- 可扩展

## 10. Phase 3 的完成标准

Phase 3 不是“写完一篇长文”就算结束。它的完成标准是：

1. 你能从文档里看出新游戏的核心循环。
2. 你能看出六线拓扑怎么变成设计骨架。

4. 你能看出 Phase 4 的代码模块怎么拆。
5. 这份设计稿能作为实际实现的输入，而不是分析笔记。
