# 大航海 II 游戏逻辑说明

这份文档不是讲解压缩、反汇编或编码细节，而是用“游戏规则”的方式说明：

1. 这个游戏的剧情和分支是怎么组织的。
2. 我们现在已经破解到哪里。
3. 哪些部分已经比较清楚，哪些部分还只是候选判断。

---

## 1. 这个游戏到底是什么结构

《大航海时代 II》不是一条固定剧情线，而是一个由很多层组成的结构：

- 6 个主人公，各自有独立故事线。
- 通用任务系统，任何主人公都可能接到。
- 港口交互系统，包含买卖、雇人、酒馆、投资、委托等。
- 剧情事件系统，决定什么时候触发对话、转场、任务推进。
- 条件分支系统，决定同一个事件在不同状态下走不同路径。

所以它不是“看一段剧情就结束”的游戏，而是“角色状态 + 地点 + 时间 + 任务进度 + 旗标”共同决定事件走向的游戏。

## 2. 我们现在已经知道的游戏逻辑

### 2.1 六条主人公线

现在已经能从文本内容看出，6 个主人公的故事方向大致是这样的：

- 贵族出身线：被逐出家门，围绕身世和传说展开。
- 军官线：偏海军、军令、职责。
- 王室任务线：和国王、宫廷任务有关。
- 探险/制图线：偏地理发现与航海探索。
- 底层/还债线：更偏港口生存、欠债、酒馆环境。
- 冒险/追梦线：更自由，围绕理想和航海展开。

这说明游戏的主线不是统一的，而是“同一套世界规则下的不同主人公剧情骨架”。

### 2.2 通用任务系统

已经确认的任务大类包括：

- 运货
- 采购
- 送信
- 剿海盗
- 讨债

这类任务通常有完整的流程：

1. 任务发布。
2. 玩家接受或拒绝。
3. 任务进行中会检查时间或进度。
4. 成功后领奖。
5. 超时后失败或改写结果。

也就是说，任务不是单纯一段对话，而是“对话 + 条件 + 时间限制 + 结果分支”的组合。

### 2.3 港口日常交互

港口里能做的事已经比较清楚了：

- 贸易
- 投资
- 酒馆聊天
- 招募水手
- 买船
- 补给
- 交任务

这些内容构成了游戏的“日常循环”。玩家不是只看剧情，而是在港口和航海之间不断循环推进状态。

### 2.4 发现物和道具

发现物图鉴、道具、角色名等内容都已经可读。它们的作用是：

- 发现物：支撑探索玩法。
- 道具：支撑战斗、决斗或功能系统。
- 角色名册：支撑剧情人物和世界人物关系。

这部分已经比较明确，不是当前主要卡点。

## 3. 我们现在对“分支”的理解

分支不是只指“选 A 还是选 B”的对话框。这个游戏里的分支至少有四类：

### 3.1 事件分支

同一段剧情会根据当前状态走不同路径。状态可能包括：

- 你是谁
- 现在在哪个城市
- 当前时间到了哪一天
- 某个任务有没有接
- 某个旗标有没有被点亮

### 3.2 对话分支

同一个人物在不同条件下，会说不同文本。

### 3.3 任务分支

任务有接、拒绝、进行中、成功、超时等不同结果。

### 3.4 控制流分支

这是最底层的那种分支，决定脚本下一步跳到哪里。
目前我们已经把 João 开场的一部分脚本整理成了“候选执行拓扑”，说明这里确实不是纯文本顺序，而是带控制跳转的结构。
