# 大航海 II 游戏逻辑说明

这是一份合并版说明。它不讲反汇编过程，而是直接讲目前已经恢复出的游戏逻辑、分支结构和实际成果。

---

## 1. 游戏结构总览

《大航海时代 II》不是单线剧情，而是一个状态驱动的分支系统。

它至少由这些层组成：

- 6 条主人公故事线
- 通用任务系统
- 港口交互系统
- 对话分支系统
- 事件分支系统
- 脚本控制流系统

核心意思很简单：

```text
主人公身份 + 地点 + 时间 + 任务进度 + 旗标
-> 决定当前事件怎么走
```

所以它不是“看一段剧情”，而是“在一张状态网络里不断推进”。

## 3. 六条主人公线

我们现在已经能从文本里看出 6 条主人公线的大致类型。

### 3.1 贵族出身线

特点：

- 被逐出家门
- 牵涉管家、公爵、公爵夫人
- 以身世、继承和传说推进

### 3.2 军官线

特点：

- 更偏军令、职责、调令
- 角色身份更官方

### 3.3 王室任务线

特点：

- 和国王、宫廷、监督官相关
- 更像被派发任务的路线

### 3.4 探险/制图线

特点：

- 偏地理、发现、新航路
- 更强调探索

### 3.5 底层/还债线

特点：

- 开场更贴近酒馆、欠债、生存压力
- 更像从港口底层开始

### 3.6 冒险/追梦线

特点：

- 主人公资源少
- 更强调个人目标和远航动机

### 3.7 这说明什么

这说明游戏不是一个统一主线，而是同一世界规则下的多主人公结构。

## 4. 通用任务系统

`Snr0.mes` 对应的是通用任务系统，不是某条主人公线。

已经明确的任务类型包括：

- 运货
- 采购
- 送信
- 剿海盗
- 讨债

典型任务流程是：

1. 发布任务
2. 接受或拒绝
3. 任务进行中
4. 成功完成
5. 超时失败

这说明任务不是一段话，而是“文本 + 条件 + 时间限制 + 结果分支”的组合。

## 7. 分支结构

这个游戏里的分支至少有四种。

### 7.1 事件分支

同一个事件会根据当前状态走不同路径。状态可能包括：

- 你是谁
- 在哪座城市
- 当前时间
- 某个任务是否接下
- 某个旗标是否触发

### 7.2 对话分支

同一个人物在不同条件下会说不同文本。

### 7.3 任务分支

任务会分成：

- 接
- 拒绝
- 进行中
- 成功
- 超时

### 7.4 控制流分支

这是最底层的分支，决定脚本下一步跳到哪里。

我们已经在 João 开场里看到了这一层的局部结构。

---

## 8. 事件脚本结构

现在已经确认，SNDT 脚本不是普通文本，而是脚本控制层。

当前稳定能导出的结构是：

```text
文件 -> chunk -> 子脚本 -> 派发表 -> 字节码
```

已知规模是：

- `Snr0-6.dat` 共 **48 个 chunk**
- 共 **186 个 subscript**
- 共 **745 条 dispatch edge**

这说明脚本不是顺着读完的，而是会按派发表跳到不同块。

## 10. João 开场里具体看到了什么

### 10.1 角色槽位

在这段里，`cc` 值已经能对应到一些具体角色：

- `cc=18`：公爵
- `cc=31`：老水手洛克
- `cc=32`：神父
- `cc=97`：酒馆老板娘
- `cc=98`：歌女
- `cc=0`：多为主人公自身或默认槽

这说明同一段脚本里已经有“谁在说话”的结构信息。

### 10.2 控制边

在 João 开场里，已经有一批控制操作数很像真实跳转边，例如：

- `fe 07 48`
- `ad 00 07 48`
- `ac 00 08 07`

这些边不是随机字节，而是经常落到已知文本块起点或块内位置。

### 10.3 这说明什么

说明剧情不是线性文本，而是“对话块 + 控制边 + 状态选择”组成的网络。

## 11. 现在已经明确到什么程度

### 11.1 已经比较确定

- 游戏有多主人公结构
- 每条主人公线有不同剧情骨架
- 通用任务系统独立存在

- SNDT 脚本承载事件流程
- João 开场已经能画出带控制边的局部拓扑图

### 11.2 已经不是猜测

下面这些已经不是空推测：

- 文本编号和剧情文本能对应起来
- 任务文本有完整流程
- 派发表边确实存在
- motif 记录能对应到真实文本
- João 开场的局部结构不是线性文本，而是带分支关系的网络

## 12. 还没完全确认的部分

还没有完全确认的主要是：

- 每个 opcode 的准确语义
- 每个条件判断到底在检查什么变量
- 哪些边是真跳转，哪些只是参数传递
- 全游戏统一拓扑图
- 运行时如何精确走到某个分支

所以现在的状态是：

```text
内容层基本清楚
结构层很大一部分清楚
执行层还没完全验明
```
