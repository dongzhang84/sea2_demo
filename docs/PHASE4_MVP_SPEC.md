# Phase 4 MVP Spec

> 这是从 [PHASE3_DESIGN_DOC.md](/Users/dong/Projects/sea2_demo/docs/PHASE3_DESIGN_DOC.md:1) 压出来的可实现最小规格。
>
> 原则：先做一个能跑、能分叉、能回流的小世界，不先铺全量内容。

---

## 1. MVP 目标

MVP 不是复刻原作，也不是把六线全做完。

MVP 只验证一件事：

```text
一个港口世界可以因为人物、地点、资源和状态变化而自己分叉。
```

### 必须证明的体验

1. 玩家能在港口接到人物驱动的任务。
2. 玩家能做贸易、修船、出航。
3. 玩家能看到状态变化影响后续事件。
4. 玩家能在两条桥节点之间来回穿。

---

## 2. MVP 范围

### 世界范围

- 3 个港口
- 2 条可航线路
- 1 个海上中继点
- 1 个贸易循环
- 1 个事件回流点

### 人物范围

只保留能证明“交叉拓扑”的人物。

- `公爵夫人`
- `约翰`
- `卡特琳娜`
- `梅尔卡特`
- `法布利斯`
- `夏洛克`
- `皮耶德`
- `劳拉`
- `萨达姆`
- 主角本人

### 主线范围

优先做这 3 段：

1. `Snr1` 起点
2. `Snr5` 资助 / 寻宝桥
3. `Snr6` 贸易 / 发家桥

理由：

- `Snr1` 提供起步压力和身份变化。
- `Snr5` 提供资助、目标和寻宝。
- `Snr6` 提供经济循环、借债、回流。

这三段已经能构成一个完整闭环。

---

## 3. MVP 的核心循环

```text
接人物任务
-> 去港口整理资金 / 货物 / 船
-> 航行到下一港口
-> 触发事件 / 交易 / 轻战斗
-> 更新状态
-> 打开下一段桥节点
```

### 玩家每轮管理的状态

- 资金
- 船只
- 船员
- 声望
- 当前港口
- 任务旗标
- 关键人物关系

### 玩家每轮做的决定

- 接不接委托
- 去哪个港口
- 先贸易还是先修船
- 是否继续追桥节点

---

## 4. 必须实现的系统

### 4.1 港口系统

- 港口列表
- 港口状态
- 贸易价格
- 航行连接

### 4.2 贸易系统

- 3 到 5 种商品
- 买入 / 卖出
- 价格随港口与状态变化

### 4.3 航行系统

- 出航
- 到港
- 简单风向或航路成本

### 4.4 事件系统

- 事件触发条件
- 事件结果
- 状态推进
- 事件回流

### 4.5 人物系统

- 人物出现
- 人物状态
- 人物桥节点
- 对话结果

---

## 5. 事件结构

MVP 里的每个事件都应该是这种结构：

```yaml
event:
  id: snr1_intro_01
  actor: duchess
  location: duke_house
  trigger:
    - money_below: 1
    - ship_ready: false
  effects:
    - set_flag: funding_started
    - unlock: port_trade
    - add_relation: duchess
```

### 事件类型

- 起点事件
- 资助事件
- 航行事件
- 贸易事件
- 状态回流事件

## 6. 数据 schema

### 世界数据

```yaml
world:
  ports:
    - id: lisbon
      name: 里斯本
      goods: []
      routes: []

  routes:
    - from: lisbon
      to: next_port
      cost: 1
      wind_sensitive: true
```

### 人物数据

```yaml
characters:
  - id: duchess
    role: bridge
    line: snr1_snr5
    states: []
```

### 状态数据

```yaml
state:
  money: 0
  fame: 0
  ship_level: 0
  flags: []
  current_port: lisbon
```

---

## 7. 画面范围

MVP 只需要 4 个界面：

1. 港口主界面
2. 贸易界面
3. 航行界面
4. 事件对话界面

不做：

- 大世界全地图
- 全海战
- 全角色立绘系统
- 全主线剧情

---

## 8. 代码拆分

建议实现拆成这些模块：

- `world/`
- `trade/`
- `travel/`
- `events/`
- `characters/`
- `ui/`

如果用 Godot 4，第一版只需要：

- 一个主场景
- 一个港口场景
- 一个对话场景
- 一个交易场景

---

## 9. 完成标准

MVP 算完成，不是“内容很多”，而是下面这些都成立：

1. 玩家能从一个港口出发。
2. 玩家能赚钱或亏钱。
3. 玩家能触发一条桥节点。
4. 玩家能看见状态变化影响后续事件。
5. 玩家能从一个桥节点走到另一个桥节点。

---

## 10. 不做什么

MVP 阶段不做：

- 六条主线全量剧情
- 全地图
- 全船战斗
- 全历史事件
- 复杂 AI 船队

这不是缩水，而是先把“分叉世界”跑通。

---

## 11. 一句话版

**Phase 4 MVP = 3 港口 + 3 段桥节点 + 贸易循环 + 航行循环 + 状态回流。**
